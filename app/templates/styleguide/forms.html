Forms

All form inputs are responsive out of the box. Form elements, including labels, are styled automatically, so no need to add any special classes to them. You can put them inside grid columns for an inline or stacked look. To make the element take up a whole column, use the u-full-width class.

Buttons

You can style any element such as <a>, <button>, <input> or <submit> by giving it a button class.

Anchor Element 
<a class="button">Anchor Element</a>
<button>Button Element</button>
<input type="button" value="Input Button">
<input type="submit" value="Submit Button">

Adding button-primary class will turn it into a primary button that is styled to be more prominent.

Anchor Element 
<a class="button button-primary">Anchor Element</a>
<button class="button-primary">Button Element</button>
<input type="button" class="button button-primary" value="Input Button">
<input type="submit" class="button button-primary" value="Submit Button">

<button> and <input> elements with disabled parameter will automatically get styled accordingly. <a> tag has to have button-disabled class to get styled.

Anchor Element 
<a class="button button-disabled">Anchor Element</a>
<button class="button-primary" disabled>Button Element</button>
<input type="button" class="button button-primary" value="Input Button" disabled>
<input type="submit" class="button button-primary" value="Submit Button" disabled>

Text Input

No additional classes are necessary to style <input> elements.

        <!-- Standard text input -->

               class="u-full-width" 
               placeholder="Placeholder">

        <input type="password"

Textarea

<textarea> can have all the standard cols and rows properties to control it's size. Alternatively, you can use the u-full-width class to make it take up an entire column.

<label for="textarea-demo">Textarea Demo</label>
<textarea id="textarea-demo" cols="32" placeholder="Textarea placeholder"></textarea>

Radio & Checkboxes

Radio buttons and checkboxes are rendered by putting the <input> tag inside the <label> tag. This is done so clicking on the label can activate the radio/checkbox.

Bool Checkbox

There a way to style checkboxes that need more prominent status, usually reserved for boolean variables like Remember Me questions. The checkbox and label both need to be in a <div> container with class checkbox-container, then the checkbox itself has to be inside a <div> with a checkbox class. Additionally, a special empty <div> with class checkbox-replacement has to be below the checkbox <input> tag itself. Here's an example of what it looks like:

<!-- Main container for the whole component -->
<div class="checkbox-container">
    <!-- Container for the checkbox itself -->
    <div class="checkbox">
        <input type="checkbox" id="bool-checkbox-demo" value="y">
        <!-- Special empty div to hold the replacement checkbox -->
        <div class="checkbox-replacement"></div>
    </div>
    <!-- Label for the checkbox -->
    <label for="bool-checkbox-demo">Bool Checkbox Demo</label>
</div>

Validation

Front-end form validation is handled by jQuery Form Validator. Back-end validation is handled by Flask-WTF. In order for the front-end validation to be rendered correctly, first you'll need to include the validator library and an initialization sattement in {% block scripts %} of your template. It is not included by default because most pages never have forms and don't need validation.

Here's an example of a field that checks if input is a valid email address when the field looses focus:

<script type="text/javascript" src="{{ url_for('static', filename='jQuery-Form-Validator/form-validator/jquery.form-validator.min.js') }}"></script>
<script>
$(document).ready(function() {
    $.validate({
        form : '#form_id',
        errorMessageClass : 'danger'
    });
});
</script>

<!-- Other markup and template statments would be here -->

<form id="form_id" name="form_name" action="" method="post">

    <!-- CSRF protection -->
    {{ form.hidden_tag() }}

    <!-- Render your field -->
    {{ render_field(form.email(**{
                        'data-validation':'email', 
                        'data-validation-error-msg-container':'#form-email-error',
                        'data-validation-error-msg':'Incorrect'
                    }), 
                    form.email.label,
                    form.email.flags) }}

</form>

The render_field() macro is describe in the Form Helpers section of this style guide. A full list of possible validators is available on the jQuery Form Validator website.
